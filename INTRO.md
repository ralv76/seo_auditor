# SEO Auditor

Автоматизированный инструмент полного SEO-аудита сайта по URL. Работает без участия человека: краулит страницы, проверяет технические параметры, запускает LLM-анализ и формирует отчёт.

---

## Проблема

Ручной SEO-аудит занимает 4–8 часов специалиста. Типичный чек-лист: title, description, H1, ссылки, битые ссылки, скорость, Schema.org, дубли, изображения, тексты — всё проверяется вручную или полуавтоматически через набор разных инструментов. Результат: длительно, дорого, невоспроизводимо. Проблема критична для агентств и продуктовых команд, которым нужно регулярно оценивать качество сайтов клиентов или конкурентов.

## Пользователь

- **SEO-специалисты и аналитики** — быстрая оценка клиентского сайта
- **Маркетологи и product-менеджеры** — проверка технического состояния посадочных страниц
- **Студии и агентства** — массовый аудит портфолио клиентов
- **Технологические стартапы** — автоматическая оценка качества генерируемых сайтов

## Основная функция

| Этап | Что происходит |
|---|---|
| **Вход** | URL главной страницы сайта |
| **Краулинг** | BFS-обход: robots.txt, sitemap.xml, внутренние ссылки. Лимиты по страницам и глубине |
| **Извлечение** | Парсинг HTML: title, description, H1–H6, canonical, OG, Schema.org, alt изображений, текст, ссылки |
| **Локальный анализ** | Проверка: HTTPS, длина title/description, дубли, битые ссылки, изображения без alt, TTFB, размер страницы, редиректы, счётчики аналитики |
| **LLM-анализ** | 2 запроса к OpenAI-совместимому API: логичность структуры (title/description/H1/H2) и качество текстов |
| **Выход** | Отчёт в Markdown + HTML, raw_data.json, aggregated.json |

## Формат результата

- **report.md** — структурированный отчёт с таблицами и рекомендациями
- **report.html** — визуализированный отчёт для отправки клиенту
- **raw_data.json** — сырые данные по каждой странице
- **aggregated.json** — агрегированные метрики и проблемы
- **pages/** — сохранённые HTML-страницы (опционально)

## Минимальный функционал (MVP)

**Реализовано:**
- Краулинг до 20 страниц, глубина 3
- Локальные проверки: мета-теги, заголовки, дубли, битые ссылки, производительность, аналитика, изображения
- LLM-анализ (2 запроса: структура + тексты)
- Генерация MD и HTML отчётов
- Конфигурация через YAML
- Подключаемые плагины (Python-скрипты)

**Планируется:**
- Увеличение лимита страниц (100+)
- Многопоточный краулинг
- Интеграция с Yandex Search API, PageSpeed Insights API
- Сравнительный конкурентный аудит (два сайта рядом)
- Сравнительный аудит версий сайта (один сайт в разные даты)
- Экспорт в PDF и XLSX
- Веб-интерфейс (FastAPI) и поддержка бота
- Расширение функционала и критериев анализа текстов
- Расширение функционала и критериев анализа страниц и структуры сайта
- Расширение функционала и критериев анализа внутренних, внешних и входящих ссылок
- Улучшение структуры и картинки генерируемых отчетом в html

## Технологии

| Компонент | Стек |
|---|---|
| Язык | Python 3.11+ |
| HTTP / краулинг | `requests`, `BeautifulSoup4` |
| Конфигурация | `PyYAML` |
| Отчёты | `Jinja2` (HTML-шаблоны), `markdown` |
| LLM | OpenAI-совместимый API через `openai` SDK |
| Форматы вывода | Markdown, HTML, JSON |
